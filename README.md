Obsidian Absolute Image Transfer (绝对路径图片转换器)

这是一个专为 Obsidian 打造的本地图片本地化插件。

当你从其他软件（如 QQ、微信、Typora 等）导出或复制 Markdown 笔记到 Obsidian 时，会有两个问题：

链接失效隐患：复制粘贴产生的往往是本地物理链接（如 file:///D:\...），一旦清理聊天记录或移动文件，图片就会全部失效。

文件名乱码崩溃：某些软件（尤其是 QQ）生成的图片文件名极其“逆天”（如包含大量斜杠、百分号和特殊符号），这会导致 Obsidian 的渲染引擎直接崩溃，无法识别路径。

本插件完美解决了这些痛点：一键将笔记中的外部物理图片提取、复制到你的 Obsidian 仓库中，并自动转换为原生的双链格式。

✨ 核心特性

一键提取与转换：自动解析形如 file:/// 或 C:\ 的本地物理图片路径，将图片物理搬运至仓库内，并将链接转为 ![[Pasted image 2026xxxx.png]]。

动态智能附件目录 (v1.0.2)：完全遵照您在设置中指定的附件路径（支持完美跟随 Obsidian 系统的原生附件设置，或自定义全局/局部路径），并支持多级目录的安全自动递归创建。

强大的极端命名兼容 (Hardcore Edge Cases)：

完美兼容路径中的空格及中文字符。

独家防弹设计：通过弹性探测算法，完美解析包含乱码、 % 号混淆、甚至带有半角括号 () {} 的变态文件名，自动剥离 Obsidian 强加的转义符 \。

无损防冲突机制：采用时间戳（YYYYMMDDHHmmss）对图片进行标准化重命名，若遇到同秒并发，自动顺延探测，绝不覆盖已有文件。

🚀 快速使用

插件提供了多种灵活的使用方式，满足不同场景的需求：

1. 快捷键 & 命令面板

打开一篇包含绝对路径图片的笔记。

使用 Ctrl/Cmd + P 呼出命令面板，搜索 转换当前笔记中的外部图片 并执行。

2. 右键菜单 (Context Menu) —— 最强推荐！

处理单篇笔记：在左侧文件树中，右键点击任意 .md 文件，选择 转换本文件内的外部图片 或 重命名本文件内的乱码图片。

批量处理文件夹：右键点击任意文件夹，选择 转换该文件夹下的外部图片，插件将自动遍历该文件夹及其所有子文件夹，一键完成迁移。

3. 全局暴力清理

在命令面板中执行 转换整个仓库中的外部图片，一键处理整个仓库中的所有笔记。

📦 安装说明

手动安装

从本仓库的 Releases 页面下载最新版本的 main.js 和 manifest.json。

在你的 Obsidian 仓库中，进入 .obsidian/plugins/ 目录。

新建一个名为 obsidian-absolute-image-transfer 的文件夹并放入上述文件。

在 Obsidian 设置 -> 第三方插件中开启本插件。

🎉 v1.0.2：原生附件存储体系深度融合

现在插件拥有了独立的设置面板，全面接管附件路径分配：

5 种存储策略：

跟随系统设置 (默认)：图片去哪，完全听从 Obsidian 系统设置里的“附件默认存放路径”。

仓库根目录 / 当前文件所在目录。

指定子文件夹 / 指定全局文件夹：支持自定义名称（如 Assets/Images）。

递归目录生成：指定不存在的多级深层目录时，插件会自动创建层级，告别写入报错。

🎉 v1.0.1：库内乱码图片清理

智能扫描：自动识别文件名带有 \、%、{}、() 等乱码字符的内部双链图片。

原生级重命名：重命名物理文件的同时，瞬间自动更新全库所有引用了该图片的笔记链接。

⚠️ 重要警告：严禁使用 Ctrl + Z 撤销！

由于重命名涉及物理文件的更改，按下 Ctrl + Z 只会把笔记内的文本变回乱码旧链接，但硬盘上的旧文件已不存在，会导致图片显示破损。请务必谨慎操作或提前备份仓库。
