# Obsidian Absolute Image Transfer (绝对路径图片转换器)

这是一个专为 Obsidian 打造的本地图片本地化插件。

当你从其他软件（如 QQ、微信、Typora 等）导出或复制 Markdown 笔记到 Obsidian 时，会有两个问题
1. 复制粘贴是本地文件链接的格式，如!(file:///D:\local_data\software_data\tool_data\communicate_data\QQdata\Tencent%20Files\xxx\Image\Group2\Q)\TN\Q)TNF]S%MRO@AI1(F[I]OYC.gif)，聊天记录一清空全没了。
2. 某些版本qq对文件逆天的起名，如Q)\TN\Q)TNF]S%MRO@AI1(F[I]OYC.gif，会导致Obsidian的渲染直接死掉，因为根本分不清哪个斜杠是路径，哪个斜杠是文件名，obsidian会直接无法显示。
我试过local image plus,但是因为文件名太阴间这个插件也死掉了。
用我这个插件直接右键转换，全部链接都变成附件，完美。

本插件完美解决了这个痛点：**一键将笔记中的外部物理图片提取、复制到你的 Obsidian 仓库中，并自动转换为原生的双链格式。**

## ✨ 核心特性

- **一键提取与转换**：自动解析形如 `file:///` 或 `C:\` 的本地物理图片路径，将图片物理搬运至仓库内，并将链接转为 `![[Pasted image 2026xxxx.png]]`。
- **智能附件目录**：自动在当前笔记的同级目录下寻找或创建 `Attachments` 文件夹来存放图片，保持仓库整洁。
- **强大的极端命名兼容 (Hardcore Edge Cases)**：
  - 完美兼容路径中的空格及中文字符。
  - **独家防弹设计**：完美解析包含乱码、 `%` 号混淆、甚至带有半角括号 `()` `{}` 的变态文件名（如 QQ 生成的复杂图片名），自动剥离 Obsidian 强加的转义符 `\`。
- **无损防冲突机制**：采用时间戳（YYYYMMDDHHmmss）对图片进行标准化重命名，若遇到同秒并发，自动顺延探测，绝不覆盖已有文件。

## 🚀 快速使用

插件提供了多种灵活的使用方式，满足不同场景的需求：

### 1. 快捷键 & 命令面板
- 打开一篇包含绝对路径图片的笔记。
- 按下快捷键。
- 或使用 `Ctrl/Cmd + P` 呼出命令面板，搜索 `转换当前笔记中的外部图片` 并执行。

### 2. 右键菜单 (Context Menu) 最强推荐！
- **处理单篇笔记**：在左侧文件树中，右键点击任意 `.md` 文件，选择 `转换该文件内的本地图片`。
- **批量处理文件夹**：右键点击任意文件夹，选择 `转换该文件夹内所有本地图片`，插件将自动遍历该文件夹及所有子文件夹，一键完成海量笔记的图片迁移！

### 3. 全局暴力洗地
- 在命令面板中执行 `转换整个仓库中的外部图片`，一键处理整个仓库中的所有笔记。

## 📸 转换效果演示

**转换前 (外部脆弱链接):**
```markdown
这里是一段笔记内容。
![](file:///D:\QQ_Data\Tencent%20Files\Image\58`7R\(F{BB37HFZ7$@FL%_H.png)
```

**转换后 (Obsidian 原生双链):**
```markdown
这里是一段笔记内容。
![[Pasted image 20260420123045.png]]
```
*(同时图片已被安全地复制到了当前目录的 `Attachments` 文件夹中)*

## 📦 安装说明

### 手动安装
1. 从本仓库的 [Releases](../../releases) 页面下载最新版本的 `main.js` 和 `manifest.json`。
2. 在你的 Obsidian 仓库中，进入 `.obsidian/plugins/` 目录。
3. 新建一个名为 `obsidian-absolute-image-transfer` 的文件夹。
4. 将下载的两个文件放入该文件夹中。
5. 重启 Obsidian，在设置 -> 第三方插件中关闭安全模式，并启用本插件。

### 使用 BRAT 安装 (推荐测试版用户)
1. 安装并启用 [BRAT](https://github.com/TfTHacker/obsidian42-brat) 插件。
2. 在 BRAT 设置中点击 `Add Beta plugin`。
3. 输入本仓库的地址：`KFLhoTAT/obsidian-absolute-image-transfer`。
4. 在 Obsidian 插件列表中启用它。

## ⚠️ 注意事项

- 转换过程会真实读取你电脑硬盘上的图片文件。如果原始图片在转换前已经被你从硬盘上删除，插件将跳过该链接并在控制台输出警告。
- 建议在执行全局批量处理 (`转换整个仓库中的外部图片`) 之前，先使用 Git 或 Obsidian 的同步功能备份您的仓库，以防万一。

## 📄 更新日志
### 🎉 v1.0.1 新增功能：库内乱码图片一键重命名

很多时候，从社交软件（如 QQ、微信）导入到 Obsidian 库中的图片，虽然已经是内部双链形式，但文件名往往包含大量转义符或乱码（例如：![[7A\6\\7A6`L]%69[Z3J72){%)D0CY.gif]]`）。这不仅影响了源码模式下的美观，也让附件管理变得一团糟。

在 1.0.1 版本中，我们引入了乱码双链图片清理功能！

#### ✨ 功能亮点

智能扫描：自动识别并定位当前笔记（或整个文件夹）中，文件名带有 \、%、{}、() 等非常规字符的内部双链图片。

原生级重命名：调用 Obsidian 底层 API 进行处理。它不仅会在物理硬盘上将图片重命名为干净的 Pasted image YYYYMMDDHHmmss 格式并移至 Attachments 文件夹，还会瞬间在后台更新你整个仓库中所有引用了该图片的笔记链接，彻底告别断链烦恼。

一键批量处理：支持在左侧文件树中右键点击单篇笔记或整个文件夹，选择 重命名本文件/该文件夹下的乱码图片 进行批量清理。

#### ⚠️ 重要警告：严禁使用 Ctrl + Z 撤销！

由于本插件对乱码图片的处理涉及底层物理文件的真实重命名，请在操作前务必知晓以下机制：

撤销不对等：如果您在插件执行重命名后，在当前笔记中按下 Ctrl + Z (Windows) 或 Cmd + Z (Mac) 试图撤销，Obsidian 只会撤销文本编辑器的更改（把干净的双链变回原来的乱码双链）。

导致断链：然而，硬盘上的真实图片已经被重命名了。您的笔记试图去读取旧的乱码名字，就会直接导致图片显示破损（File not found）。

如何防范：在对包含大量笔记的文件夹执行批量重命名之前，强烈建议您先使用 Git 或其他方式备份您的仓库。操作一旦执行，请将其视为永久更改，不要试图用编辑器的撤销键来回滚。
